TheRouter boasts four key capabilities:

Navigator:

  • Support Activity, Fragment, ViewController
  • Page paths support declaration using regular expressions.
  • Supports exporting route tables in JSON and plist formats.
  • Supports dynamic delivery of JSON route tables, degrading any page to H5.
  • Supports interception handling of page navigation.
  • Supports custom parsing of page parameters (e.g., parsing JSON into objects).
  • Supports navigating to Activity (Fragment) in third-party SDKs using routes.

ServiceProvider:

  • Supports cross-module dependency injection.
  • Supports custom creation rules for injection items.
  • Supports custom service interception and single-module mock debugging.
  • Supports injection object caching, with only one new object created for multiple injections.

FlowTaskExecutor:

  • Supports independent initialization of a single module.
  • Supports lazy loading initialization.
  • Independent initialization allows multiple task dependencies (similar to Gradle Task).
  • Supports compile-time circular reference detection.
  • Supports custom business initialization timing, addressing privacy compliance.

ActionManager:

  • Supports global callback configuration.
  • Supports priority response and interrupt response.
  • Supports recording call paths for debugging Observables in observer mode.

About TheRouter

TheRouter is an open-source solution framework developed by Huolala Technology, suitable for Android/iOS modular development. It has advantages such as high dynamism, remote delivery, and dual-end consistency. It not only decouples regular module dependencies and page navigation but also provides solutions to common problems in the modularization process.

Why Use TheRouter

Routing is an indispensable function in modern mobile development, especially for enterprise-level apps. It can decouple the strong dependency relationship of Intent page navigation and reduce the mutual dependency issues between teams in cross-team development.

TheRouter is a complete set of solutions dedicated to modular development, supporting both Android and iOS. It not only decouples regular module dependencies and page navigation but also provides solutions to common problems in the modularization process. It perfectly solves the problem of unable to obtain external module lifecycle and business processes within the component, resulting in the need to modify code across modules for each initialization and dependent call.
